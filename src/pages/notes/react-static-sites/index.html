What is GatsbyJS?

  • A static site generator
  • that uses GraphQL
  • ReactJS
  • and Webpack

What Gatsby Does

  • Generates static HTML files
  • and a JS bundle
  • so your site can be both
    • dynamic
    • and static
  • where you first download the initial HTML file
    • to get really fast loads
  • then for any additional information such as pulling in from an api
  • you still have access to all that React power
  • all the while generating static site files
  • meaning you get html and js files you can throw in anywhere
    • makes hosting easy
    • no database to worry about
  • makes blog posts easy because
    • you just write everything in markdown
    • then save it to a folder
    • then you can access that data using GraphQL query

First Look

Slides

Wants: How do we know this is a use case?

What static gives you

  • Indexable by search engine
  • Performant - loads quickly
  • (Relatively)Secure - there's no backend
  • Easily Deployable - no downtime, simple push

Developer Experience - all checks

  • Full power of Javascript - think UI
  • Use existing knowledge - they all have learning curves
  • write once - we just don't know how yet
  • Hot reloading, modularized code, ES6, etc - WEBPACK! - bundles and writes things to disk!

The process of what we want to do

  1. we write a plugin
  2. that gives you all the routes
  3. that then turns those into strings
  4. and match them against components
  5. and render those all out
  6. to static files

Great... how?

  1. Turn React Router consig into a list of routes
  2. Iterate over that list, match w/ React Router to match those strings
  3. Save strings (when we match a string we get a component, get the sting value using React's renderToString methods) then save that string to html files on disk
  4. Webpack is super good at writing files, a good combo w/ React Router that enables you to make nested routes
  5. see webpack plugin @ iansinnott/react-static-webpack-plugin
    • turns all your routes into static files

So what does it all look like?

  • Build front-end app
  • npm run build

    you get a static site back

  • Deploy full static website to the host of your choice
